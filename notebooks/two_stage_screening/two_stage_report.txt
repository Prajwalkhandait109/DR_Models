================================================================================
        DIABETIC RETINOPATHY DETECTION USING DEEP LEARNING
================================================================================

Architecture Name: Two-Stage Screening Pipeline
                   (EfficientNetV2B0  +  Swin Transformer)

--------------------------------------------------------------------------------
1. OVERVIEW
--------------------------------------------------------------------------------
Diabetic Retinopathy (DR) is a diabetes-induced complication that progressively
damages the blood vessels of the retina, potentially leading to permanent vision
loss. Early and accurate detection is critical for timely clinical intervention.

This project presents an automated DR screening and grading system based on a
two-stage cascade:

  STAGE 1 — Binary Screening (EfficientNetV2B0):
      Decides whether a fundus image shows ANY diabetic retinopathy or is
      healthy (No_DR vs. DR). This stage mimics the real-world clinical
      screening step where the majority of cases are healthy and must be
      filtered out efficiently.

  STAGE 2 — Severity Grading (Swin Transformer):
      Only images flagged as DR by Stage 1 are passed to a Swin Transformer
      that classifies them into one of four clinical severity grades:
      Mild, Moderate, Severe, or Proliferative DR.

This decoupled design improves overall sensitivity (Stage 1 is tuned for high
recall on DR), reduces confusion between No_DR and Mild cases at the grading
level, and provides a clinically aligned workflow. GradCAM explainability maps
are generated on Stage 2 predictions to provide visual justification for the
final severity decision.

--------------------------------------------------------------------------------
2. DATASET DESCRIPTION
--------------------------------------------------------------------------------
Source      : Kaggle — "Diabetic Retinopathy 224x224 2019 Data"
              (sovitrath/diabetic-retinopathy-224x224-2019-data)
Downloaded  : Via KaggleHub API
Image Folder: colored_images/

Original 5-class labels:
  0 — No_DR           (No signs of diabetic retinopathy)
  1 — Mild            (Microaneurysms only)
  2 — Moderate        (More than microaneurysms, less than severe)
  3 — Severe          (Extensive hemorrhages, no PDR signs)
  4 — Proliferate_DR  (Neovascularization or vitreous/pre-retinal hemorrhage)

Label Reformulation:

  Stage 1 (Binary):
      No_DR                                 → "No_DR"
      Mild / Moderate / Severe / Proliferate_DR → "DR"

  Stage 2 (4-class severity, No_DR removed):
      Mild            → 0
      Moderate        → 1
      Severe          → 2
      Proliferate_DR  → 3

Split:
  Training Set  : 80% of total images
  Validation Set: 20% of total images
  Stratified    : Yes, on the original 5-class label

DataLoader / Generator:
  Batch Size  : 32
  Num Workers : 2 (PyTorch stage)
  Shuffle     : True (train), False (validation)

3. PREPROCESSING AND AUGMENTATION
--------------------------------------------------------------------------------

  3.1 STAGE 1 — EfficientNetV2B0 (TensorFlow / Keras)
  ─────────────────────────────────────────────────────────────────────────────
  Training Augmentations (ImageDataGenerator):
    - rescale=1./255              : Normalize pixel values to [0,1]
    - rotation_range=20           : Random rotation up to ±20 degrees
    - horizontal_flip=True        : Random horizontal flipping
    - zoom_range=0.2              : Random zoom in/out by up to 20%

  Validation Transforms:
    - rescale=1./255 only (no augmentation)

  Input Size: 224 × 224 × 3 (RGB)

  3.2 STAGE 2 — Swin Transformer (PyTorch / torchvision)
  ─────────────────────────────────────────────────────────────────────────────
  Training Transforms:
    - Resize(224, 224)
    - RandomHorizontalFlip       : Simulates mirror-image fundus views
    - RandomRotation(±10°)       : Handles camera orientation variability
    - ToTensor                   : Converts PIL image to [0,1] float tensor
    - Normalize
        Mean : [0.485, 0.456, 0.406]   (ImageNet mean)
        Std  : [0.229, 0.224, 0.225]   (ImageNet std)

  Validation Transforms:
    - Resize(224, 224)
    - ToTensor
    - Normalize (same mean and std as above)

  Input Size: 224 × 224 × 3 (RGB)

4. MODEL ARCHITECTURE DESCRIPTION
--------------------------------------------------------------------------------

  4.1 STAGE 1 — EFFICIENTNETV2B0 BINARY CLASSIFIER
  ─────────────────────────────────────────────────────────────────────────────
  Base Model : EfficientNetV2B0
  Source     : tensorflow.keras.applications
  Pretrained : ImageNet
  Input      : 224 × 224 × 3 RGB

  Key Architectural Properties:
    - Compound scaling of depth, width, and resolution.
    - Fused-MBConv + MBConv blocks for fast training and efficient inference.
    - Squeeze-and-Excitation (SE) channel attention.
    - Designed for high accuracy at a low parameter / FLOP budget — ideal
      for the screening (high-throughput) stage.

  Classification Head:
    - GlobalAveragePooling2D
    - Dropout(0.3)
    - Dense(1, activation='sigmoid')   →  P(DR)

  Total Parameters: ~7 million (Base variant)
  Output          : Single probability ∈ [0, 1] for the DR class.

  4.2 STAGE 2 — SWIN TRANSFORMER SEVERITY CLASSIFIER
  ─────────────────────────────────────────────────────────────────────────────
  Base Model : Swin Transformer Base — swin_base_patch4_window7_224
  Source     : timm (PyTorch Image Models)
  Pretrained : ImageNet-21k / ImageNet-1k

  Key Architectural Properties:
    - Patch Size          : 4 × 4 pixels per token
    - Window Size         : 7 × 7 patches per attention window
    - Shifted Window      : Alternating regular and shifted windows enable
                            cross-window attention without global computation
    - Hierarchical Stages : 4 stages with patch merging (multi-scale features)
    - Embedding Dim       : 128 (Base variant)
    - Attention Heads     : [4, 8, 16, 32] per stage
    - Depth per Stage     : [2, 2, 18, 2]
    - MLP Ratio           : 4.0

  Classification Head:
    - The final classification head was replaced with a 4-class linear layer
      (Mild / Moderate / Severe / Proliferate_DR).

  Target Layer for GradCAM:
    - model.layers[-1].blocks[-1].norm1
      (Final block's normalization layer in the last Swin stage)

  Total Parameters: ~87 million (Base variant)
  Output          : 4-class logits → softmax → argmax for prediction

  4.3 PIPELINE FUSION (INFERENCE)
  ─────────────────────────────────────────────────────────────────────────────
  predict_dr(img_path):
      1. EfficientNetV2B0 produces P(DR).
      2. If P(DR) < 0.3   →   return "No_DR"   (high-recall screening).
      3. Else, Swin Transformer predicts one of:
             {Mild, Moderate, Severe, Proliferate_DR}
         and that label is returned.

  The low threshold (0.3) biases Stage 1 toward sensitivity — preferring
  false positives at screening over missing real DR cases.

5. TRAINING STRATEGY
--------------------------------------------------------------------------------

  5.1 STAGE 1 — EFFICIENTNETV2B0 (Two-Phase)
  ─────────────────────────────────────────────────────────────────────────────
  Phase 1: Head-Only Training
      Epochs       : 5
      Trainable    : Dense head only (base_model.trainable = False)
      Optimizer    : Adam (default LR = 1e-3)
      Loss         : binary_crossentropy
      Metrics      : accuracy, Recall (sensitivity), AUC

  Phase 2: Fine-Tuning
      Epochs       : 10
      Trainable    : All layers (base_model.trainable = True)
      Optimizer    : Adam, LR = 1e-5
      Loss         : binary_crossentropy
      Metrics      : accuracy, Recall (sensitivity), AUC

  Saved Artifact : efficientnetv2b0_binary_dr.h5

  5.2 STAGE 2 — SWIN TRANSFORMER (Two-Phase)
  ─────────────────────────────────────────────────────────────────────────────
  Phase 1: Head-Only Training
      Epochs       : 5
      Trainable    : Classification head only (backbone frozen)
      Optimizer    : Adam, LR = 1e-3
      Loss         : CrossEntropyLoss
      Purpose      : Warm up the randomly initialized 4-class head without
                     disturbing pretrained ImageNet weights.

  Phase 2: Full Fine-Tuning
      Epochs       : 15  (Total Stage 2 epochs: 20)
      Trainable    : All parameters unfrozen
      Optimizer    : Adam, LR = 1e-5 (low LR avoids catastrophic forgetting)
      Loss         : CrossEntropyLoss
      Purpose      : Gradually adapt all layers to the retinal imaging domain.

  Checkpointing:
    - Full checkpoint (model + optimizer)  : swin_checkpoint.pth
    - Model weights only                   : swin_transformer_dr.pth
    - Complete pickled model               : complete_swin_model.pth

  Device: CUDA (GPU) if available, else CPU

6. EVALUATION METRICS
--------------------------------------------------------------------------------

  6.1 STAGE 1 METRICS (BINARY)
  ─────────────────────────────────────────────────────────────────────────────
  Tracked during training via Keras:
    - Accuracy
    - Recall / Sensitivity   (critical for medical screening)
    - AUC                    (overall discrimination of DR vs. No_DR)

  6.2 STAGE 2 ACCURACY
  ─────────────────────────────────────────────────────────────────────────────
  Computed as the fraction of correctly classified images over the Stage 2
  validation subset (No_DR samples removed).

      Accuracy = Correct Predictions / Total Predictions

  Validation accuracy was tracked per epoch across all 20 Stage 2 epochs.

  6.3 CONFUSION MATRIX (STAGE 2)
  ─────────────────────────────────────────────────────────────────────────────
  A 4×4 matrix showing the count of predictions per true class vs. predicted
  class. Visualized as a heatmap using seaborn with 'Blues' colormap.

  Rows    : Actual class labels (Mild / Moderate / Severe / Proliferate_DR)
  Columns : Predicted class labels
  Diagonal: Correct classifications

  Output: notebooks/two_stage_screening/output/confusion_matrix.png

  6.4 ROC-AUC CURVES (STAGE 2)
  ─────────────────────────────────────────────────────────────────────────────
  One-vs-Rest (OvR) ROC curves computed for each of the 4 severity classes
  using softmax probabilities. AUC reported per class.

  Method      : sklearn roc_curve + auc
  Binarization: label_binarize for multi-class probability extraction

  Output: notebooks/two_stage_screening/output/roc_auc.png

  6.5 CLASSIFICATION REPORT (STAGE 2)
  ─────────────────────────────────────────────────────────────────────────────
  Per-class breakdown of:
    - Precision : TP / (TP + FP)
    - Recall    : TP / (TP + FN)
    - F1-Score  : 2 × (Precision × Recall) / (Precision + Recall)
    - Support   : Number of ground-truth samples per class

  Generated using sklearn.metrics.classification_report.
  Output: notebooks/two_stage_screening/output/classification_report.txt

  6.6 GRADCAM EXPLAINABILITY (STAGE 2)
  ─────────────────────────────────────────────────────────────────────────────
  Gradient-weighted Class Activation Mapping (GradCAM) was applied to the
  Swin Transformer to visualize which regions of the fundus image influenced
  each severity prediction.

  Target Layer : model.layers[-1].blocks[-1].norm1
  Library      : pytorch-grad-cam
  Overlay      : show_cam_on_image() blends heatmap over original RGB image

  Two sets of GradCAM outputs:
    Correct Predictions: One sample per class where True == Predicted
    Wrong Predictions  : One sample per class where True != Predicted

  Output folder: notebooks/two_stage_screening/output/gradcam/
    ├── correct/   (GradCAM on correctly classified samples)
    └── wrong/     (GradCAM on misclassified samples)

7. STRENGTHS OF THE PROPOSED SYSTEM
--------------------------------------------------------------------------------

  7.1 TECHNICAL STRENGTHS
  ─────────────────────────────────────────────────────────────────────────────
  - Cascaded Specialization: Stage 1 specializes in the easier
    "healthy vs. diseased" decision while Stage 2 specializes in fine-grained
    severity grading. Each model focuses on what it does best.

  - Reduced Class Imbalance for Stage 2: Removing the dominant No_DR class
    before training the Swin Transformer alleviates the heavy imbalance in
    APTOS 2019 and improves minority-class learning.

  - Best-of-Both-Worlds Architectures: A lightweight CNN (EfficientNetV2B0)
    handles high-throughput screening; a powerful hierarchical transformer
    (Swin) handles complex severity discrimination.

  - High-Sensitivity Screening: A low decision threshold (0.3) on Stage 1
    favors recall, in line with medical screening priorities where missed
    positives are far costlier than false alarms.

  - Two-Phase Fine-Tuning per Stage: Both models use head-only warm-up
    followed by full fine-tuning at a low LR — a stable strategy that avoids
    destroying pretrained ImageNet features.

  - Explainability via GradCAM: Provides visual evidence for each Stage 2
    prediction, making the transformer interpretable — critical for clinical
    deployment and auditing.

  7.2 MEDICAL RELEVANCE
  ─────────────────────────────────────────────────────────────────────────────
  - Clinically Aligned Workflow: Mirrors real-world DR screening — first
    detect presence of disease, then grade severity if disease is present.

  - Standard Grading Output: The 4-class Stage 2 output maps directly to the
    International DR Clinical Classification Scale (excluding No_DR).

  - GradCAM Localization: Heatmaps highlight lesion-associated regions such
    as hemorrhages, exudates, and neovascularization — aligning with clinical
    diagnostic markers.

  - Early Detection Capability: Mild and Moderate stages are explicitly
    modeled, enabling early intervention before vision-threatening progression.

  - Screening Support: High-throughput automated screening can assist in
    resource-limited settings where ophthalmologists are scarce.

  7.3 MODEL LIMITATIONS
  ─────────────────────────────────────────────────────────────────────────────
  - Error Propagation: A false negative at Stage 1 (DR misclassified as
    No_DR) can never be recovered by Stage 2. Overall recall depends heavily
    on Stage 1 sensitivity.

  - Fixed Stage 1 Threshold: The 0.3 cutoff is hand-picked, not learned or
    validated on a held-out set. Calibration (e.g., via ROC analysis) would
    yield a more principled operating point.

  - No Class Weighting in Stage 2: Even after removing No_DR, Stage 2 still
    has imbalance (especially under-represented Severe / Proliferative
    classes). No weighted loss or resampling was applied.

  - Limited Augmentation: Augmentations are basic (flip, small rotation,
    zoom). Retina-specific augmentations (brightness, contrast, CLAHE,
    color jitter) were not used, which may reduce cross-device generalization.

  - Fixed Input Resolution: Both stages require 224×224 input. High-resolution
    fundus images must be downsampled, potentially losing fine lesion detail
    relevant to early DR.

  - No Learning Rate Scheduler: Neither stage uses a scheduler (e.g.,
    CosineAnnealingLR or ReduceLROnPlateau), which may lead to suboptimal
    late-epoch convergence.

  - Mixed Frameworks: Stage 1 runs in TensorFlow and Stage 2 in PyTorch.
    This complicates deployment (two runtimes) and increases serving overhead.

  - No Independent Test Set: Evaluation is performed only on a stratified
    validation split from the same distribution. An external held-out test
    set is needed for unbiased performance reporting.
